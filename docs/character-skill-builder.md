# 人物 Skill 生成模块设计文档

## 1. 文档定位

本文档定义 `StoneStory-skill` 中人物 skill 第一版生成模块的设计目标、输入输出约定、证据分层思路与后续演进方向。

本文档关注的是“人物 skill 资产如何由输入数据生成”，不负责说明：

- 角色对话运行时如何消费这些文件
- 最终回答生成时的 prompt 编排
- 大模型推理链路如何组织

## 2. 设计目标

人物 skill 第一版的目标不是直接生成完整的人物画像，而是先建立一条“证据优先”的资产生成链路。

当前模块重点解决：

- 从 `triples.csv` 中抽取人物直接关系
- 从 `passages.jsonl` 中抽取人物相关原文证据
- 对原文证据进行基础去噪和相关性打分
- 从去噪结果中提炼更适合做说话风格分析的证据子集

当前版本明确不做：

- 直接写无证据支撑的人物性格结论
- 直接自动生成最终角色口吻
- 直接自动生成完整对话示例

## 3. 输入与输出

### 3.1 输入

当前输入文件：

- `skill/characters/*/*.skill/manifest.json`
- `data/input/triples.csv`
- `data/output/passages.jsonl`

输入职责：

- `manifest.json`
  - 提供人物基础标识、别名、证据包含词、排除词
- `triples.csv`
  - 提供人物直接关系三元组
- `passages.jsonl`
  - 提供原文段落级证据候选

### 3.2 输出

当前每个人物 skill 目录下输出：

- `relations.md`
- `evidence_passages.jsonl`
- `evidence_ranked.jsonl`
- `style_evidence.jsonl`
- `source_report.json`
- `persona.md`
- `style.md`
- `examples.md`

其中：

- `relations.md`
  - 用于记录直接关系事实
- `evidence_passages.jsonl`
  - 用于记录人物相关原始证据候选
- `evidence_ranked.jsonl`
  - 用于记录去噪和相关性打分后的证据
- `style_evidence.jsonl`
  - 用于记录更适合分析人物表达风格的证据子集
- `source_report.json`
  - 用于记录本次生成的计数和数据来源

## 4. 三层证据链路

当前人物 skill 的证据生成不是一步完成，而是拆成三层。

### 4.1 第一层：原始证据

输出文件：

- `evidence_passages.jsonl`

生成方式：

- 逐条扫描 `passages.jsonl`
- 用人物 `manifest.json` 中的 `evidence_terms` 去匹配段落正文
- 若文本中命中人物名、别名或其他显式证据词，则收录为原始证据
- 若命中内容只落在 `exclude_terms` 对应短语中，则视为误命中，不收录

这一层解决的问题是：

- 哪些段落与该人物直接相关

这一层不解决的问题是：

- 哪些段落更重要
- 哪些段落更适合分析说话风格

### 4.2 第二层：去噪与打分

输出文件：

- `evidence_ranked.jsonl`

生成方式：

- 对第一层原始证据逐条打分
- 同时记录增强信号 `signals`
- 同时记录噪声标记 `noise_flags`

当前已使用的信号包括：

- 命中人物全名
- 命中别名
- 同时命中多个识别词
- 包含引语符号
- 包含人物发话标记
  - 例如 `宝玉道`
  - 例如 `黛玉笑道`
- 包含人物在场信号
  - 例如 `宝玉来了`

当前已使用的噪声标记包括：

- 只命中短别名
- 无对话信号
- 包含易混淆对象名

这一层解决的问题是：

- 哪些人物相关段落更值得优先关注
- 哪些段落虽然相关，但质量较弱或噪声较大

注意：

- 当前版本是“打分排序”，不是“硬过滤”
- 所以 `evidence_ranked.jsonl` 目前仍保留全部原始证据，只是附加了分数和噪声标记

### 4.3 第三层：风格证据提炼

输出文件：

- `style_evidence.jsonl`

生成方式：

- 从第二层去噪结果中继续筛选
- 优先保留带有人物发话信号或明确对话上下文的段落
- 当前要求证据至少具备一定相关性分数

当前已识别的风格信号类型包括：

- `speech_marker`
  - 段落中出现人物名与发话标记组合
- `named_dialogue_context`
  - 段落带有引语且明确出现人物名

这一层解决的问题是：

- 哪些段落适合拿来分析人物如何说话
- 哪些段落可作为后续 `style.md` 的依据

这一层仍不直接做的事：

- 直接写人物风格结论
- 直接生成最终角色回答样例

## 5. 为什么要分三层

分层的核心原因是三件事并不等价：

- 人物相关
- 人物相关且较高质量
- 可用于分析人物表达方式

也就是说：

- 第一层只回答“有没有关系”
- 第二层回答“这条证据质量怎么样”
- 第三层回答“这条证据能不能拿来研究说话风格”

如果不做分层，容易出现两个问题：

- 仅仅提到人物名字的段落被直接拿去写风格
- 角色设定重新退回到主观印象总结

## 6. 与当前角色文件的关系

当前人物 skill 目录里的几个文件应这样理解：

- `relations.md`
  - 关系事实文件
- `persona.md`
  - 数据约束版人物说明
  - 当前只记录可确认的信息和约束，不写主观画像
- `style.md`
  - 风格分析约束说明
  - 当前主要说明证据基础和筛选规则
- `examples.md`
  - 示例生成约束说明
  - 当前不自动生成具体角色对话示例

所以目前这套设计的重点是：

- 先把“证据基础”建起来
- 再在这个基础上做更严格的人物风格提炼

## 7. 当前限制

当前实现仍有这些限制：

- 原始证据仍偏多，去噪还只是打分，不是正式过滤
- 风格证据仍是规则筛选，不是稳定的人物语言建模结果
- 证据相关性主要依赖字符串匹配，尚未做更细的说话人识别
- `triples.csv` 中若存在名称错误或脏数据，会直接影响关系输出

## 8. 后续演进方向

建议后续按下面顺序演进：

1. 为去噪结果引入正式过滤阈值
2. 增加人物发话识别规则，降低“人物在场但并未发话”的误收录
3. 为风格证据增加更细的标签
   - 情绪表达
   - 称呼方式
   - 对不同人物的语气差异
4. 基于风格证据生成可回溯的 `style.md` 正式结论
5. 基于证据生成更可信的角色回答示例

## 9. 优化思路

当前版本采用的是规则驱动 baseline。

这里的 baseline 含义是：

- 先用最容易解释、最容易复现、最容易调试的方案跑通链路
- 当前并不声称该方案已经优于 embedding 或模型打分
- 当前也没有完成严格的离线效果对比

后续优化可以沿三个方向展开。

### 9.1 规则优化

适用目的：

- 继续提升可解释性
- 快速修正规则边界
- 低成本迭代

可做方向：

- 为 `evidence_ranked.jsonl` 引入硬过滤阈值
- 增加更多误命中排除规则
- 增加更细的发话标记与说话人识别规则
- 为不同人物单独维护高风险混淆词

优点：

- 成本低
- 可控性强
- 便于人工核查

限制：

- 泛化能力弱
- 对复杂语义判断支持有限

### 9.2 Embedding 检索优化

适用目的：

- 提升人物相关段落召回率
- 找到不直接出现人物名、但语义上高度相关的段落

可能做法：

- 为 `passages.jsonl` 构建向量索引
- 用人物名、别名、关系词、场景词组成查询向量
- 对召回结果再接人工规则或模型重排

潜在收益：

- 比纯字符串匹配更容易找到隐式相关段落
- 对别名变化、近义表达更友好

限制：

- 召回更强不等于更精准
- 结果可解释性弱于规则
- 需要额外引入 embedding 依赖与索引构建链路

### 9.3 模型打分 / 模型重排优化

适用目的：

- 提升证据精排质量
- 判断某段是否真正适合做人物风格证据

可能做法：

- 对第一层召回结果做 LLM rerank
- 让模型判断：
  - 该段是否真在刻画这个人物
  - 该段是否适合作为风格证据
  - 该段是否更像背景叙述而非人物语言证据

潜在收益：

- 比纯规则更接近语义判断
- 有机会更准确地区分“人物相关”与“风格相关”

限制：

- 成本更高
- 稳定性弱于固定规则
- 如果缺少评测集，容易产生“看起来更聪明，但无法验证是否更准”的问题

## 10. 对比评估思路

当前仓库尚未完成规则方案、embedding 方案和模型打分方案的正式对比实验。

因此当前不能得出以下结论：

- 规则一定优于 embedding
- embedding 一定优于规则
- 模型打分一定优于规则

当前更准确的表述是：

- 规则方案是当前第一版 baseline
- embedding 和模型打分是后续可验证的优化方向
- 是否更优，需要通过统一评测来判断

建议的对比实验方式如下。

### 10.1 先建立小规模人工标注集

建议先选：

- 贾宝玉
- 林黛玉

每个人物各抽样一批段落，由人工标注为以下类别：

- 无关
- 相关但弱
- 高质量人物证据
- 适合做风格证据

### 10.2 对比的候选方法

第一组：

- 规则 baseline

第二组：

- embedding 召回
- 规则重排

第三组：

- embedding 召回
- 模型打分 / 模型重排

### 10.3 评估指标

建议至少比较：

- Top-K 命中率
- 高质量证据 precision
- 风格证据 precision
- 人工可解释性
- 构建成本
- 稳定性与可复现性

### 10.4 评估原则

- 同一批标注样本必须用于所有方案
- 召回和重排要分开看，不混成一个指标
- 除了效果，还要看解释成本和维护成本

## 11. 当前结论边界

当前可以成立的结论只有：

- 规则方案已经能作为第一版可运行 baseline
- 当前链路已经实现了从人物关系到原文证据、再到风格证据的分层构建
- 这套方案适合当前阶段的低依赖、可解释、可迭代目标

当前不能直接成立的结论包括：

- 规则方案效果最好
- embedding 或模型方案一定更好
- 当前风格证据已经足够支撑最终人物口吻生成
